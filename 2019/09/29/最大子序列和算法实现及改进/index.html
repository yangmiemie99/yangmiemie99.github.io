最大子序列和算法实现及改进

发表于 2019-09-29 | 分类于 算法

一 描述

给定一个整数数组 nums ，找到一个具有最大和的连续子数组（子数组最少包含一个元素），返回其最大和。

示例:

输入: [-2,1,-3,4,-1,2,1,-5,4],
输出: 6
解释: 连续子数组 [4,-1,2,1] 的和最大，为 6。

二 实现

下面介绍本题的三种解法,规律,分而治之,动态规划.

1 最开始我的的想法

利用规律,忽略对子序列进行寻找比较,和是正数就可以一直向后加,如果不是就重新确定一个头是整数的最大子序列起点,复杂度O(n),空间复杂度O(1),使用了两遍循环,效果差一些,,下面写出一遍循环解决.

2 改进的做法

经过学习,进行改进,思路基本相同不过把负数的例外情况在一个循环中体现出来了.性能基本不变,这个也可以说用了动态规划的思想,不过不明显.

3 利用分而治之的思想

利用分而治之对这个问题而言性能没有上边的算法好,作为练习,写代码.时间复杂度O(NlogN),将问题分为两个大致相等的子问题,然后用递归对他进行求解,这是分的部分.然后将两个子问题修补到一起,并可能在做一些少量的 附加工作.

在这个例子中,最大子序列可能出现在三处,整个输入数据的左半部分,或者右半部分,或者横跨左右两部分.

三者最大的就是最大子序列.

4 动态规划

任何数学递推公式都可以直接翻译成递归算法.但是基本现实是:编译器常常不能正确的对待递归算法,结果导致低效的程序.当我们怀疑这种情况时,我么需要给编译器一点帮助,将递归算法重新写成非递归算法,让后者把这些子问题的答案记录在一个表内.利用这种技巧的方法叫做动态规划.

动态规划的两个步骤是思考“状态”和“状态转移方程”

搅动太规划划分成三步:

  • base case : 思考问题规模最小时,是什么情况
  • update function: 自下而上的思考这个问题,即上面的“状态转移方程”
  • gola: 终点强调输出是什么,很多时候输出不一定是最后一个状态

对本题进行分析

1 状态

题目中哦个有关键字连续,首先我们尝试将状态定义乘题目要求的结果:即dp[i]表示nums在区间[0,i]中连续子数组的最大和.在思考“状态转移方程时”,dp[i]之前的,例如dp[i - 1]就有可能是更前面的连续子数组的最大和,这两者之间的关系比较复杂,不利于写出状态空间方程.

既然一个连续的子数组一定要以一个数作为结尾,那么我没问你就将状态定义为

dp[i] : 表示以 nums[i] 结尾的连续子数组的最大和

注意:

如果状态的定义不是题目中为题的定义,一定要考虑输出是什么,不能将最后的一个状态返回出去.

输出应该是把所有的dp[0],dp[1],dp[2]...dp[n-1]都看一遍取最大值

2 状态转移方程

dp[i]的值要看dp[i -1],因为nums[i]一定被寻取那么dp[i-1]的政府就成为分类的标准

  • 如果 dp[i - 1] >= 0，那么可以把 nums[i] 直接接在 dp[i - 1] 表示的那个数组的后面。

  • 如果 dp[i - 1] < 0，那么加上前面的数反而我越来越小了，干脆“另起炉灶”吧，单独的一个 nums[i]，就是 dp[i]。
